# Twitter Bookmarklets

A small website that distributes bookmarklets for cleaning up your Twitter account.

The list of available tools lives on the site itself and may grow over time. The site UI and bookmarklet runtime code are written in English.

## Features

- Drag-and-drop install on desktop browsers
- One-click copy of the `javascript:` bookmarklet URL for phones
- Setup guides for computer and phone
- Safety confirm dialog before destructive actions start
- Soft stop support via `window.__twitterCleanStop = true`

## Requirements

- [Bun](https://bun.sh/) 1.3+
- A modern browser (Chrome, Edge, Firefox, or Safari)
- A Twitter account and access to the pages each bookmarklet targets

## Getting started

```bash
bun install
bun run dev
```

Then open the local URL printed by Vite (usually `http://localhost:5173`).

### Scripts

| Command              | Description                                           |
| -------------------- | ----------------------------------------------------- |
| `bun run dev`        | Start the Vite development server with HMR            |
| `bun run ogp`        | Regenerate OGP images (`summary` + `summary_large`)   |
| `bun run build`      | Type-check with TypeScript, then build for production |
| `bun run deploy`     | Build and deploy to Cloudflare Workers                |
| `bun run deploy:dry` | Build and dry-run the Workers deploy                  |
| `bun run cf:dev`     | Build and preview with Wrangler                       |
| `bun run preview`    | Serve the production build locally                    |
| `bun run lint`       | Run Oxlint                                            |
| `bun run fmt`        | Format with Oxfmt                                     |
| `bun run fmt:check`  | Check formatting with Oxfmt                           |

## How to install the bookmarklets

### On a computer

1. Show the bookmarks bar.
   - Chrome / Edge: `Ctrl+Shift+B` (Windows/Linux) or `Cmd+Shift+B` (macOS)
   - Safari: enable **Favorites Bar** from the View menu
2. Open this site.
3. Drag a bookmarklet button onto the bookmarks bar.
4. Open the matching Twitter page, then click the bookmark to run it.

Clicking the button on this site does nothing on purpose. It only works as a bookmark target.

### On a phone

1. Tap **Copy script** for the bookmarklet you want.
2. Create a new bookmark (or edit an existing one).
3. Replace the bookmark URL with the copied `javascript:...` string.
4. Save it, open the matching Twitter page, then run the bookmark.

Some mobile browsers make editing bookmark URLs awkward. If pasting fails, try another browser or finish the setup on a computer first.

## Stopping a run

- Reload the page, or
- In the browser console, run:

```js
window.__twitterCleanStop = true;
```

## Warnings

- Destructive actions **cannot be undone**.
- These bookmarklets automate UI clicks on Twitter. They may break when Twitter changes its DOM or `data-testid` attributes.
- Use them only on your own account and at your own risk.
- Running them too quickly or for long periods may trigger rate limits or other account restrictions.

## Development notes

Bookmarklet source lives in `src/bookmarklets.ts`. The exported `href` values are generated by stripping comments and collapsing whitespace into a single `javascript:` URL.

The OGP images are committed assets regenerated with `bun run ogp`:

- `public/ogp.png` — `summary_large_image` (1200×630, icon + title)
- `public/ogp-summary.png` — `summary` (800×800, icon only)

For production SEO (absolute Open Graph URLs, sitemap, and robots Sitemap), the default site URL is `https://twitter-bookmarklets.com`. Override it when needed:

```bash
SITE_URL=https://staging.example bun run build
```

Or set `VITE_SITE_URL` for the same absolute URLs in the client. The sitemap is generated by [`vite-plugin-sitemap`](https://www.npmjs.com/package/vite-plugin-sitemap).

## Deploy (Cloudflare Workers)

This site is an assets-only Worker (not Pages). SPA routes are handled with `assets.not_found_handling = "single-page-application"`.

```bash
bun run deploy
```

The Worker is named `twitter-bookmarklets` and binds the custom domain `twitter-bookmarklets.com` in `wrangler.jsonc`.

When you add or change a bookmarklet:

1. Edit the English source in `src/bookmarklets.ts`
2. Keep confirm/alert copy clear about irreversibility
3. Verify drag-and-drop and copy still produce a valid `javascript:` URL
4. Test manually on the matching Twitter page
